# 重複イベント横並び表示 設計書

## 概要

同じ時間帯に複数のイベントが存在する場合、現在は1件しか表示されない。
Google Calendar 風の横並び（最大2列）表示に対応し、重複時も両方のイベントを確認・操作できるようにする。

## スコープ

- **対象**: 時間付きイベント（時間スロット部分）のみ
- **対象外**: 終日イベント（変更なし）

---

## データ構造・ロジック変更（`src/app/mod.rs`）

### 新規フィールド

```rust
pub struct AppState {
    // 既存フィールド...
    pub overlap_focus: u8,  // 重複時のフォーカス列インデックス（0 or 1）
}
```

### 新規メソッド

```rust
/// 指定日・指定時刻に重なるイベントを全件返す
/// イベントの datetime_start〜datetime_end が hour:00〜hour:59 と1分でも重なれば対象
pub fn events_overlapping_hour(&self, date: NaiveDate, hour: u32) -> Vec<&NotionEvent>
```

### 変更メソッド

```rust
/// overlap_focus を参照して操作対象イベントを1件返す
pub fn event_at_cursor(&self) -> Option<&NotionEvent>
```

## レンダリング変更（`src/ui/week_view.rs`）

各日カラムの時間スロットセルで重複件数をチェックし、以下のルールで描画する：

| 重複数 | 表示 |
|--------|------|
| 0件 | 空白 |
| 1件 | 現在と同じ（全幅） |
| 2件 | 左半分にイベントA、右半分にイベントB |
| 3件以上 | 左半分にイベントA、右半分に `+N`（N = 残り件数） |

- `overlap_focus` に対応する列をカーソルハイライト色で強調表示
- タイトルは列幅に合わせてトリミング

---

## キー操作（ノーマルモード）

| キー | 動作 |
|------|------|
| `←` | 重複イベントがある時間帯でフォーカスを左列（0）へ |
| `→` | 重複イベントがある時間帯でフォーカスを右列（1）へ |
| `e` | `overlap_focus` が指すイベントを編集フォームで開く |
| `dd` | `overlap_focus` が指すイベントを削除確認モードへ |

**制約**: 重複イベントがない時間帯では `←/→` は無効（誤操作防止）。
`j/k` で別の時間帯に移動した際、`overlap_focus` は 0 にリセット。

---

## 変更しないもの

- 終日イベントの表示
- フォームモード・確認モードの動作
- 既存のキーバインド（`h/l/H/L/j/k/t/Tab/n/q`）

---

## テスト方針

- `events_overlapping_hour()`: 重複あり・なし・境界値（終了時刻ちょうどは重複しない）
- `event_at_cursor()`: `overlap_focus` 0/1 それぞれで正しいイベントを返す
- `overlap_focus` リセット: `j/k` 移動後に 0 に戻る
