# Design: 15-Minute Time Slot Granularity

## Overview

週ビューの時間スロットを現在の「1行 = 1時間」から「1行 = 15分」に変更する。
これにより以下の2つの問題を同時に解決する：

1. `bar` スタイルで異なるイベントが視覚的につながって見える問題
2. 1時間以上のイベントが1スロットしか占有せず、持続時間が表現できない問題

---

## Architecture

### 単位変更: 時間 → 15分スロット

- 1日 = 96スロット（0〜95）
- スロット番号 `s` の時刻: `hour = s / 4`, `minute = (s % 4) * 15`
- `scroll_offset` の単位を **時間 → 15分スロット** に変更

---

## Component Changes

### `src/app/mod.rs`

- `scroll_offset: u16` の意味を「15分スロット番号」に変更
- 初期値: `28`（= 7 * 4、07:00）
- `scroll_up`: 1減算（= 15分上へ）
- `scroll_down`: 1加算（= 15分下へ）、上限 `88`（= 22 * 4、22:00）

### `src/ui/week_view.rs` — `render_time_slots`

**時間ラベル列：**
```
08:00   ← hour境界のみ表示
        ← :15/:30/:45 は空白
```
- スロット `s` で `s % 4 == 0` なら `HH:00` を表示、それ以外は空白行

**イベント描画ロジック：**

各スロット `s` について、そのスロットに重なるイベントを探す：
- 条件: `event.datetime_start <= slot_time < event.datetime_end`
- **開始スロット**（`slot_time == datetime_start` の時刻と一致）:
  - `bar`: `▌ タイトル`
  - `block`: 背景色 + タイトル
  - `text`: 太字テキスト
- **継続スロット**（開始以外で重なっている）:
  - `bar`: `▌`（タイトルなし）
  - `block`: 背景色のみ（空白文字）
  - `text`: 何も表示しない

**gap の自然解消：**
- 例: 10:00〜11:00 と 12:00〜13:00 のイベントがある場合
  - 11:00〜11:45（4スロット）が自然に空白になり、視覚的な分離が生まれる
- `bar` スタイルでもイベント間に空白スロットが生まれるため、つながって見える問題が解消

---

## Scroll Behavior

| 操作 | 変更前 | 変更後 |
|------|--------|--------|
| `j` | +1時間 | +15分（+1スロット）|
| `k` | -1時間 | -15分（-1スロット）|
| 初期位置 | offset=7（07:00） | offset=28（07:00）|
| 上限 | offset=22 | offset=88（22:00）|

---

## Time Label Format

```
06:00



07:00   ← scroll_offset=28 でここが先頭



08:00
```

`:15`, `:30`, `:45` 行は空白にすることで、視覚的な重さを抑えつつ15分粒度を表現する。

---

## Out of Scope

- 15分未満の粒度
- イベントの重なり（同時刻に複数イベントがある場合のカラム分割）
- スクロール単位の設定ファイル化
